## 🔧 System Requirements

### Required
- **R 3.6+** (3.6, 4.0, 4.1, 4.2 all supported)
- **Python 3.7+**
- Internet connection (for initial package installation)

### Automatically Handled
- R packages (all 11 installed automatically)
- Python packages pdfplumber and openpyxl

### Optional
- For PDF comparison feature:
  - macOS: `brew install poppler`
  - Linux: `apt-get install poppler-utils`
  - Then: `Rscript -e "install.packages(c('pdftools','png'))"`

## 💡 Typical Workflow

### Option 1: Fresh Start with PDFs
```
1. Start app
2. Go to "PDFs laden" tab
3. Upload PDF file(s)
4. Click "Tabellen extrahieren"
5. Monitor progress → status display
6. Switch to "Tabellen & Vorschau"
7. Select table → choose style/language
8. Click "Aktive Tabelle (RTF)" to download
```

### Option 2: Use Existing Tables
```
1. Start app
2. Go to "Tabellen & Vorschau"
3. Select table from list
4. Choose style/language
5. Download as RTF
```

### Option 3: Batch Processing
```
1. Go to "Batch-Export" tab
2. Select multiple tables
3. Choose style/language
4. Click "ZIP herunterladen"
5. Receive ZIP file with all tables
```

## 🐛 Troubleshooting

### App Won't Start
```R
# Check R packages manually
Rscript -e "pacman::p_load(shiny, readxl, flextable, officer, dplyr, stringr, DT, tools, htmltools, zip, colourpicker)"

# Check Python
python3 --version

# Check Python packages
python3 -c "import pdfplumber; import openpyxl; print('OK')"
```

### PDFs Won't Extract
- Check that `extract_tables.py` is in the same directory
- Verify PDF file is valid and readable
- Check dependency status in "PDFs laden" tab
- Look for error messages in status display

### Python Packages Failed to Install
```bash
# Manual installation
pip3 install pdfplumber openpyxl

# Or with pip
pip install pdfplumber openpyxl
```

### File List Not Updating
- Manually click on "Tabellen & Vorschau" tab
- Refresh browser if needed
